Overview

This project is oriented towards intermediate Arduino developers with a basic understanding of Arduino and C++. Through this game, we will be going through a mix of multiple components such as LEDs and Switches.

Prerequisites

  • Arduino Uno
  • LED
  • Switches
  • Resistors
  • Programming Basics

List Of Components

  • Arduino Uno Board
  • 16 M - M Wires
  • 5 Switches
  • 4 LED Lights
  • 4 220Ω Resistors
  • 5 10kΩ Resistors

How does it work?

Simon Says is a game where the player needs to memorize a sequence of 4 lights and repeat it once it is complete. When a user presses the start button, it will generate a sequence of 4 blinking lights that will be played. Once the sequence is completed, the user will need to repeat the sequence in order by pressing on the switches. When the user has input the wrong sequence, it will blink fast. They will have 3 lives before the game is over.

Set Up

Using a red and black wire, connect it from the 5 volt (5V) pin to the positive row and the ground (GND) pin to the negative row of the breadboard.

Afterward, put 5 push buttons along the middle of the breadboard like in the picture with 3 pins separating each one of them.

We will now set up the button to start the game, which will be the first button. Connect a wire from the A4 pin to the right leg of the button. Then, with a 10k ohm resistor, connect it under the right leg to the positive row of the breadboard. Finally, connect a wire from the left leg of the button to the negative row on the breadboard.

Once that is done, we will be setting up the LEDs and buttons to run the game. Using a wire, connect it from the 2nd digital pin to the second socket on a column to the right of the button. Then, using a 220 ohm resistor, connect it horizontally from the third socket of the same row. With an LED, connect it from the end of the resistor to the right leg of the button. From the right leg of the button, connect a wire to the negative row of the breadboard. After, connect a wire from the 1st Analog pin (A1) to the left leg of the button, and a 10k ohm resistor from that wire to the positive row of the breadboard.

Tip

LED's current goes from the long leg to the short leg, from the Anode to the Cathode.

Repeat this process with every button left.

Coding

First, you will need to declare your light and switch pins.

const int switch1 = A0;
const int switch2 = A1;
const int switch3 = A2;
const int switch4 = A3;

const int start_switch = A4;

const int light1 = 5;
const int light2 = 4;
const int light3 = 3;
const int light4 = 2;

Tip: Why not hard code the values as we code?

This is a safe practice that allows the pins to be set to specific numbers. When you need to change the pins, you can refer to these values instead of changing them individually. It can become messy if you have a lot of code.

Inside the void setup() function, you can set the pin modes for the LEDs and switches as such:

void setup() {
  pinMode(switch1, INPUT);
  pinMode(switch2, INPUT);
  pinMode(switch3, INPUT);
  pinMode(switch4, INPUT);

  pinMode(light1, OUTPUT);
  pinMode(light2, OUTPUT);
  pinMode(light3, OUTPUT);
  pinMode(light4, OUTPUT);
}

Once that is done, you can set the pinMode for each light to LOW to have the lights off when the project starts.

void setup() {
  pinMode(switch1, INPUT);
  pinMode(switch2, INPUT);
  pinMode(switch3, INPUT);
  pinMode(switch4, INPUT);

  pinMode(light1, OUTPUT);

  pinMode(light4, OUTPUT);

  digitalWrite(light1, LOW);
  digitalWrite(light2, LOW);
  digitalWrite(light3, LOW);
  digitalWrite(light4, LOW);
}

Certain attributes are needed for this game to work. For example, we will need the sequence size, an array of integers to store a generated sequence, an array of integers to store the sequence input, an integer value for the number of lives the player has, an integer value for the number of lives left, and a boolean value to indicate if the game is started or not. At the beginning of the code, right after where we defined the lights and switches, we will declare the following attributes.

const int sequence_size = 4;
int sequence[sequence_size];
int input_sequence[sequence_size];

bool started = false;
const int chances = 3;
int errors = 0;

Inside the void loop(), we will be checking if the has pressed the start button to begin the game, if the player has exceeded their number of errors to end the game and to generate the sequence and wait for the sequence if the game is running.

void loop()
{
  if (errors == chances)
  {
     errors = 0;
    error_repetition = 3;
    started = false;
  }
  if (digitalRead(start_switch) == LOW && started == false)
  {
    started = true;    
  }
  if (started == true && errors < chances)
  {
    set_sequence();
    get_sequence();
  }
}

As you can see inside the loop, we have two undefined functions: get_sequence() and set_sequence(). We will start with the set_sequence() function.

In this function, we will be generating 4 random numbers, that correspond to the 4 lights available using randomSeed() and millis().

The randomSeed() function comes from the Arduino board that initializes the pseudo-random number generator at a random point in time.

The millis() function, also a function from the Arduino board, returns the number of milliseconds passed since the board began running the current program.

Afterward, we will use a For Loop to generate 4 numbers with the random() function and store them inside the sequence attribute created earlier.

void set_sequence()
{
  randomSeed(millis());

  for (int i = 0; i < sequence_size; i++)
  {
    sequence[i] = random(2,6);
  }
}

Once that is done, we will make sure that the lights are all off. Then, we will display the sequence using a for loop that runs through the sequence attribute.

void set_sequence()
{
  randomSeed(millis());

  for (int i = 0; i < sequence_size; i++)
  {
    sequence[i] = random(2,6);
  }

  digitalWrite(light1, LOW);
  digitalWrite(light2, LOW);
  digitalWrite(light3, LOW);
  digitalWrite(light4, LOW);

  for (int i = 0; i < sequence_size; i++)
  {
    digitalWrite(sequence[i], HIGH);
    delay(1000);
    digitalWrite(sequence[i], LOW);
    delay(200);
  }
}

The next function that we will need to write is the get_sequence() function that allows the board to get the input sequence from a user.

This function will have a flag attribute, that by default will be set to 0, meaning the board is awaiting input from the player. Once a player has pressed on a switch, the flag will be set to 1 and whatever the user has input wouldn't be registered until the system has processed the verification.

A For Loops is needed to get exactly the number of inputs indicated at the beginning under the sequence_size attribute.

Inside the For Loop, we will put a While Loop. While the Flag is 0, it will be waiting for input from one of the four switches.

There will be an If statement, that will check if one of the switches is pressed. If so, it will light up the corresponding LED, insert into the input_sequence the triggered switch, set the flag to 1, and check if the input_sequence[i] is not equal to the sequence[i].

If the sequence is wrong, it will run the wrong_sequence() function which is a sequence of lights to indicate that the sequence is wrong. It will also add 1 to the attribute errors and break the For Loop.

If the sequence is right, it will continue the loop until it is done. Once all the sequences have been checked, it will run the right_sequence() function which is a sequence of lights to indicate that the sequence is right.

void get_sequence()
{
  int flag = 0;

  for (int i = 0; i < sequence_size; i++)
  {
    flag = 0;
    while(flag == 0)
    {
      if (digitalRead(switch1) == LOW)
      {
        digitalWrite(light1, HIGH);
        input_sequence[i] = 5;
        flag = 1;
        delay(200);
        if (input_sequence[i] != sequence[i])
        {
          wrong_sequence();
          errors++;
          return;
        }
        digitalWrite(light1, LOW);
       }

      if (digitalRead(switch2) == LOW)
      {
        digitalWrite(light2, HIGH);
        input_sequence[i] = 4;
        flag = 1;
        delay(200);
        if (input_sequence[i] != sequence[i])
        {
          wrong_sequence();
          errors++;
          return;
        }
        digitalWrite(light2, LOW);
       }

      if (digitalRead(switch3) == LOW)
      {
        digitalWrite(light3, HIGH);
        input_sequence[i] = 3;
        flag = 1;
        delay(200);
        if (input_sequence[i] != sequence[i])
        {
          wrong_sequence();
          errors++;
          return;
        }
        digitalWrite(light3, LOW);
      }

      if (digitalRead(switch4) == LOW)
      {
        digitalWrite(light4, HIGH);
        input_sequence[i] = 2;
        flag = 1;
        delay(200);
        if (input_sequence[i] != sequence[i])
        {
          wrong_sequence();
          errors++;
          return;
        }
        digitalWrite(light4, LOW);
      }
    }
  }
  right_sequence();
}

Here is the wrong_sequence() function. In this function, we will be blinking the lights rapidly at 200 milliseconds intervals 3 times. And it adds 3 every time the user makes a mistake. In the end, it will blink 9 times, as the user can only have 3 mistakes.

void wrong_sequence()
{
  for (int i = 0; i < error_repetition; i++)
  {
      digitalWrite(light1, LOW);
      digitalWrite(light2, LOW);
      digitalWrite(light3, LOW);
      digitalWrite(light4, LOW);
      delay(200);
      digitalWrite(light1, HIGH);
      digitalWrite(light2, HIGH);
      digitalWrite(light3, HIGH);
      digitalWrite(light4, HIGH);
      delay(200);
      digitalWrite(light1, LOW);
      digitalWrite(light2, LOW);
      digitalWrite(light3, LOW);
      digitalWrite(light4, LOW);

  }
  error_repetition = error_repetition + 3 ;
  delay(1000);
}

Here is the right_sequence() function. In this function, we will have the lights light up for half a second, and then off.

void right_sequence()
{

    digitalWrite(light1, LOW);
    digitalWrite(light2, LOW);
    digitalWrite(light3, LOW);
    digitalWrite(light4, LOW);
    delay(500);

    digitalWrite(light1, HIGH);
    digitalWrite(light2, HIGH);
    digitalWrite(light3, HIGH);
    digitalWrite(light4, HIGH);
  
  delay(500);
    digitalWrite(light1, LOW);
    digitalWrite(light2, LOW);
    digitalWrite(light3, LOW);
    digitalWrite(light4, LOW);
    delay(1000);
}
